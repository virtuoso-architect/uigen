## 1. Principios de Desarrollo
* **DRY (Don't Repeat Yourself):** Si una lógica de cálculo de precio se usa en tres lugares, crear una sola función/tool centralizada.
* **KISS (Keep It Simple, Stupid):** No complicar el código de n8n si una función simple de JavaScript lo resuelve mejor.
* **Comentarios con Sentido:** El código debe ser autodescriptivo, pero los comentarios deben explicar el "Por qué" se tomó una decisión técnica.
